# 智能门禁系统需求规格说明书

本文档描述智能门禁系统在用户管理、认证、设备控制、告警、审计和可靠性方面的需求。

## 1. 范围

系统用于办公园区的门禁控制，支持网页管理端、移动端和门禁控制器协同工作。

## 2. 用户与权限

- 管理员应能够创建、停用和恢复普通用户账号。
- 系统应记录用户账号状态变更的操作者、时间和原因。

## 3. 认证与会话

用户输入正确账号密码后，系统应完成身份验证并建立会话。

连续五次登录失败时，系统应锁定该账号十五分钟并提示用户稍后重试。

用户连续三十分钟无操作时，系统应自动结束当前会话。

## 4. 设备控制

- 授权用户刷卡成功后，门禁控制器应在一秒内释放门锁。
- 未授权凭证尝试开门时，系统应拒绝开门并记录事件。

## 5. 告警与通知

门被强行打开时，系统应在五秒内向安保人员发送告警通知。

门禁控制器离线超过两分钟时，系统应在管理端显示设备离线状态。

## 6. 审计与报表

管理员按时间范围查询开门记录时，系统应返回匹配的事件列表。

| 报表类型 | 生成要求 |
| --- | --- |
| 日报 | 系统应在每天零点生成前一日通行统计报表。 |
| 异常报表 | 系统应支持导出强行开门和未授权开门事件。 |

## 7. 性能与可靠性

在网络正常且控制器在线时，系统应在两秒内完成开门请求处理。

> 当主认证服务不可用时，系统应切换到备用认证服务。

```text
配置项 audit.retention_days 应不小于 180。
```
